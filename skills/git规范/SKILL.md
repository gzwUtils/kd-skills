# Git 规范

本技能为 Java Spring Boot 项目提供全面的 Git 版本控制最佳实践和工作流程，确保代码历史清晰、易于管理。

## 核心功能

### 分支策略
- **主分支（Main）**：生产环境就绪的代码
- **开发分支（Develop）**：功能集成分支
- **功能分支（Feature）**：用于开发单独的功能
- **发布分支（Release）**：用于准备发布版本
- **热修复分支（Hotfix）**：用于紧急缺陷修复

### 提交规范
- **提交信息格式**：统一的提交信息结构
- **提交粒度**：小而聚焦的提交
- **提交频率**：定期提交
- **提交信息**：清晰且具有描述性

### 合并请求（Pull Request）流程
- **代码审查**：合并前必须进行代码审查
- **CI/CD 检查**：自动化测试和检查
- **合并策略**：压缩合并、变基合并或普通合并
- **分支清理**：删除已合并的分支

### Git 工作流程
- **功能开发**：功能分支工作流
- **发布流程**：发布分支工作流
- **热修复流程**：热修复分支工作流
- **持续集成**：CI/CD 集成

### 虚拟电厂专项
- **设备管理**：设备相关变更使用功能分支
- **能源交易**：交易逻辑使用独立分支
- **监控模块**：监控功能使用专属分支
- **用户管理**：用户相关变更使用专属分支

## 提交信息格式

```
<类型>(<Jira-ID>): <提交说明>

```

**类型**：feat（新功能）、fix（修复）、docs（文档）、style（格式）、refactor（重构）、test（测试）、chore（构建/工具）
**Jira-ID**：任务ID
**描述**：变更的简要说明

## 工作流程示例

1. **创建功能分支**：`git checkout -b feature/device-management`
2. **开发功能**：实现功能代码
3. **提交变更**：`git commit -m "feat(device): 新增设备注册接口"`
4. **推送分支**：`git push origin feature/device-management`
5. **创建合并请求**：提交 Pull Request 进行代码审查
6. **审查并合并**：审查代码后合并至开发分支
7. **删除分支**：`git branch -d feature/device-management`

## 最佳实践

- **原子提交**：每次提交只解决一个问题
- **有意义的信息**：提交信息应清晰、具有描述性
- **分支命名**：遵循统一的分支命名规范
- **代码审查**：合并前进行充分的代码审查
- **CI/CD 集成**：配置自动化测试和检查
- **Git 钩子**：使用 Git 钩子进行提交前检查
