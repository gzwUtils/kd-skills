# 数据库设计与 SQL 评审（MySQL 8）

## 适用范围
- MySQL 8.x + MyBatis/MyBatis-Plus + Spring Boot 3.x

## 设计原则（表结构）
- **命名统一**：表/字段使用可读的业务名；避免缩写歧义；时间字段明确语义（`created_at/updated_at`）。
- **主键策略**：优先单列主键；分布式用雪花/UUID 时关注索引与写放大。
- **字段类型**：用最小可用类型（`BIGINT`/`INT`/`DECIMAL`/`VARCHAR`）；金额类避免 `FLOAT/DOUBLE`。
- **可空性**：能 `NOT NULL` 就不要 `NULL`；默认值要有业务意义。
- **约束**：唯一约束、外键（谨慎）、检查约束（必要时）明确数据质量边界。

## 索引与查询原则
- **基于访问路径建索引**：where 条件、排序、分组、join 关键列优先。
- **复合索引**：遵循最左前缀；避免“一个字段一个索引”的指数膨胀。
- **避免全表扫描**：分页/筛选必须可命中索引；大表禁止无条件查询。
- **避免 SELECT ***：明确字段集合，降低 I/O 与变更风险。
- **分页策略**：深分页用延迟关联/基于游标（seek method）替代 `offset`。

## SQL 评审清单（PR/上线前）
- [ ] SQL 是否可命中预期索引（给出 `EXPLAIN` 关键字段：type/rows/extra）？
- [ ] 是否存在隐式类型转换导致索引失效？
- [ ] Join 是否可控（驱动表选择、join 条件、结果集膨胀）？
- [ ] 是否有 N+1 查询风险（循环内查询/逐条更新）？
- [ ] 变更是否包含数据迁移与回滚方案（DDL/DML 分离、灰度）？

## 迁移与治理（建议）
- **迁移工具**：Flyway/Liquibase（统一 DDL 版本、可回溯）。
- **分离变更**：DDL 与 DML 分批执行，降低锁表风险。
- **变更评估**：大表 DDL 评估锁、复制延迟、回滚成本（必要时走 Online DDL 工具）。
