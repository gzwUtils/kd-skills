# CI/CD 与质量门禁（CI/CD Engineer）

## 适用范围
- Java 17 / Maven 或 Gradle / Spring Boot 3.x

## 流水线分层（建议）
1. **快速反馈（PR）**：编译 + 单测 + 静态检查（分钟级）
2. **增强验证（主干/合并）**：集成测试 + 组件测试 + 依赖/漏洞扫描
3. **发布验证（发布分支/标签）**：回归集 + 性能基线 + 产物签名/制品追踪

## 质量门禁（建议）
- **格式/规范**：Checkstyle/Spotless（可选）
- **静态分析**：SpotBugs/PMD/ErrorProne/Sonar（择一或组合）
- **依赖治理**：SBOM + CVE 扫描（OWASP Dependency-Check/Trivy 等）
- **测试门禁**：关键模块覆盖率阈值（不要“一刀切”，按风险分层）

## 发布与回滚
- **灰度发布**：按比例/按租户/按开关灰度
- **可回滚**：数据库变更可回退（或具备向前兼容的回滚策略）
- **制品可追溯**：构建号/提交号/环境/配置版本可追踪

## PR 规则（建议）
- [ ] 必须通过 CI
- [ ] 必须代码审查（至少 1-2 人）
- [ ] 禁止直接推主干（保护分支）
- [ ] 变更说明/影响面/回滚方案在 PR 描述中可见
